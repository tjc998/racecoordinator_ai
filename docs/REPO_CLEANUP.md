## 1. Screendiff snapshots no longer tracked in git

**What changed**

- Added to `.gitignore`:
  ```
  client/src/app/components/**/*_screendiff_test.ts-snapshots/
  ```
- `git rm --cached` of all 517 tracked snapshot PNG/JPEG files under that path.
- The files are **still on disk** — `--cached` only removes them from the index.
  The next screendiff test run will regenerate them in place; they just won't
  be staged.

**Why**

These are Playwright visual-regression baselines. They are large (~375 MB)
binary artifacts that get regenerated by
`./scripts/run_client_screendiff_tests.sh --update-snapshots`. Versioning them
inflates the repo size on every clone and creates massive diffs whenever a
component's render shifts by a pixel. They are test artifacts, not source.

**What you may want to do next**

If you want to also reclaim local disk space, delete the snapshot directories
from your working tree. The next `--update-snapshots` run will recreate them:

```bash
find client/src/app/components -type d -name '*_screendiff_test.ts-snapshots' -exec rm -rf {} +
```

(Skip this on machines that need to compare against the old baselines locally.)

### Path rewrites applied

Because the scripts moved one level deeper, every "find the repo root" hop
inside them was updated:

- **Shell scripts**
  - `cd "$(dirname "$0")/client"` → `cd "$(dirname "$0")/../client"`
  - `cd "$(dirname "$0")/server"` → `cd "$(dirname "$0")/../server"`
  - `source "$(dirname "$0")/scripts/test_env.sh"` → `source "$(dirname "$0")/test_env.sh"`
  - `node "$(dirname "$0")/scripts/sync_snapshots.js"` → `node "$(dirname "$0")/sync_snapshots.js"`
  - `run_all_tests.sh` and `run_client_tests.sh` previously invoked sibling
    scripts via `./other_script.sh`, which resolved against the caller's CWD.
    They now use an absolute `"$SCRIPT_DIR/other_script.sh"` so they keep
    working regardless of where they're invoked from.
  - `release.sh` had `cd server` (CWD-dependent). Now does
    `cd "$(dirname "$0")/.."` first.
- **PowerShell scripts**
  - In `run_*.ps1`, `$PSScriptRoot` was being used to mean "repo root". That
    is now one level up; introduced a `$RepoRoot = Split-Path -Parent $PSScriptRoot`
    (or `$ScriptDir` / `$ProjectRoot` pair) and substituted accordingly.
  - In `run_client_screendiff_tests.ps1`, the `sync_snapshots.js` lookup
    changed from `Join-Path $ProjectRoot "scripts" "sync_snapshots.js"` to
    `Join-Path $ScriptDir "sync_snapshots.js"` since the JS lives next to
    the PS1 now.
- **`check_i18n.js`**: `path.join(__dirname, 'client/...')` →
  `path.join(__dirname, '..', 'client/...')`.

## 4. Installer files moved into `scripts/installer/`

| Old path | New path |
| --- | --- |

| `installer_offline_legacy.iss` | `scripts/installer/installer_offline_legacy.iss` |

### Path rewrites applied

- `create_installers.sh` cds to the repo root at the top
  (`cd "$(dirname "$0")/../.."`). Every relative path inside (`client`,
  `server`, `release`, `build_cache`, `scripts/install_dependencies_mac.sh`)
  still resolves identically.
- `create_installers.ps1` defines
  `$RepoRoot = (Resolve-Path (Join-Path $PSScriptRoot "..\..")).Path`,
  does `Set-Location $RepoRoot`, and substitutes `$PSScriptRoot` → `$RepoRoot`
  in every spot where it meant repo root (build paths, `Set-Location`,
  `Join-Path $PSScriptRoot $TargetDir`, etc.). The two `iscc` invocations
  now pass `"$PSScriptRoot\installer_online.iss"` (and the offline variant)
  explicitly so iscc finds the scripts in their new location.
- `installer_base.iss` adds `SourceDir=..\..` to `[Setup]`. Inno Setup
  resolves `Source:` paths in `[Files]` against `SourceDir`, and `SourceDir`
  itself is resolved relative to the `.iss` file. `..\..` from
  `scripts\installer\` is the repo root, so `Source: "release\..."` paths
  still find the build outputs.

### Behavior change to be aware of

By default Inno Setup writes the compiled `.exe` to an `Output\` directory
next to the `.iss` file. That used to be `<repo>\Output\`; it is now
`<repo>\scripts\installer\Output\`. Nothing in `create_installers.ps1`
depends on the `Output\` path, so this only matters when you go looking for
the built installer. If you want the old location back, add
`OutputDir=..\..\Output` to `installer_base.iss` (path is resolved relative
to the `.iss` file).

  baselines. Three changes restored a coherent model:

  gained `OutputDir=Output`, which resolves against the new `SourceDir=..\..`
  and lands compiled installers at `<repo>\Output\` — the pre-cleanup

- **Shell script safety.**
  - `scripts/run_all_tests.sh`: dropped `set -e`; it conflicted with the

  un-tracked them; the directories remained on disk. A `find … -type d -name
  '*_screendiff_test.ts-snapshots' -exec rm -rf {} +` reclaimed the space.
  Visual tests regenerate baselines on next run.

  because its content was duplicated (lines 1–97 ≈ 98–237). Truncated to a

  Playwright / Karma blocks logically, and globbed lint/checkstyle reports
  via `**/*lint_report*.txt` / `**/checkstyle_report*.txt`. Verified the
  rewrite still catches every path the old file did.
- **PowerShell `$PSScriptRoot` idiom unified** across seven scripts. The
  three test scripts (`run_client_unit_tests.ps1`, `run_client_screendiff_tests.ps1`,
  `run_server_tests.ps1`) had a defensive three-branch fallback chain whose
  two non-`$PSScriptRoot` branches were dead in practice; replaced with the
  same `Split-Path -Parent $PSScriptRoot` form already used by the run/server
  scripts. `scripts/installer/create_installers.ps1` swapped `Resolve-Path +
  Join-Path` for `Split-Path -Parent (Split-Path -Parent $PSScriptRoot)` —
  same result, no filesystem call, consistent with siblings.
